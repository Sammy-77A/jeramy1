Authentication

Lipana uses a dual key system: publishable keys for client-side use and secret keys for server-side operations. All API requests require authentication using one of your API keys.

Dual Key System
Publishable Key

Safe to expose in client-side code, mobile apps, or public repositories. Use for frontend integrations.

lip_pk_live_...
Secret Key

Keep private and secure. Only use in server-side code. Never expose in client-side code or public repositories.

lip_sk_live_...

API Key Expiration & Session Timeout

API keys do not expire. Once created, your API keys remain valid until you manually revoke or delete them. This makes them ideal for long-running integrations, background services, and automated systems.

Session timeout does not affect API usage. The 15-minute session timeout only applies to web dashboard users who authenticate with JWT tokens (cookies). API key authentication is completely separate and is not subject to any timeout restrictions.

Note: If you're using the web dashboard, your session will expire after 15 minutes of inactivity or 15 minutes from login. You'll need to log in again. However, any API calls made with your API keys will continue to work regardless of your dashboard session status.

STK Push

Initiate M-Pesa STK (Sim Tool Kit) push payments directly via API. Send payment requests to customers' phones and receive payments instantly.

⚠️
Minimum Transaction Amount

All transactions must have a minimum amount of Ksh 10. Transactions below this amount will be rejected to prevent payout failures.

How STK Push Works

STK Push allows you to send payment requests directly to a customer's phone. When you initiate an STK push:

You send a request with the customer's phone number and amount
An M-Pesa prompt appears on the customer's phone
The customer enters their M-Pesa PIN to complete the payment
You receive a webhook notification when the payment is completed

Webhook Signature Verification

All webhooks from Lipana include an X-Lipana-Signature header that you can use to verify the authenticity of the request. This ensures that the webhook is actually coming from Lipana and hasn't been tampered with.

📋 Getting Your Webhook Secret

Your webhook secret can be found in your dashboard under Webhooks settings. Keep this secret secure and never expose it in client-side code.

How It Works

Lipana generates an HMAC-SHA256 signature of the webhook payload using your webhook secret. The signature is sent in the X-Lipana-Signature header. You should verify this signature on your server before processing the webhook.

Verification Steps
Extract the X-Lipana-Signature header from the incoming request
Get the raw request body as a string (before JSON parsing)
Compute HMAC-SHA256 of the body using your webhook secret
Compare the computed signature with the header value using constant-time comparison
If they match, the webhook is authentic. If not, reject the request with a 401 status.
Security Best Practices
Always verify signatures: Never process webhooks without verifying the signature first
Use constant-time comparison: Use timing-safe comparison functions to prevent timing attacks
Store secrets securely: Keep your webhook secret in environment variables, never in code
Get raw body: Use the raw request body (before JSON parsing) for signature verification
Reject invalid requests: Return 401 Unauthorized for any webhook with invalid or missing signatures
Log verification failures: Log failed verification attempts for security monitoring

import requests
import json

def create_payment_link():
    url = 'https://api.lipana.dev/v1/payment-links'
    headers = {
        'x-api-key': 'lip_sk_live_YOUR_SECRET_KEY',
        'Content-Type': 'application/json'
    }
    data = {
        'title': 'Premium Subscription',
        'description': 'Monthly premium subscription',
        'amount': 5000,
        'currency': 'KES',
        'allowCustomAmount': False,
        'successRedirectUrl': 'https://yourwebsite.com/success'
    }
    
    try:
        response = requests.post(url, json=data, headers=headers)
        response.raise_for_status()
        result = response.json()
        print(f'Payment link created: {result["url"]}')
        return result
    except requests.exceptions.RequestException as error:
        print(f'Error: {error}')
        raise

import (
    "bytes"
    "encoding/json"
    "fmt"
    "io/ioutil"
    "net/http"
)

type PaymentLink struct {
    Title              string `json:"title"`
    Description        string `json:"description"`
    Amount             int    `json:"amount"`
    Currency           string `json:"currency"`
    AllowCustomAmount  bool   `json:"allowCustomAmount"`
    SuccessRedirectURL string `json:"successRedirectUrl"`
}

func createPaymentLink() error {
    url := "https://api.lipana.dev/v1/payment-links"
    
    paymentLink := PaymentLink{
        Title:              "Premium Subscription",
        Description:        "Monthly premium subscription",
        Amount:             5000,
        Currency:           "KES",
        AllowCustomAmount:  false,
        SuccessRedirectURL: "https://yourwebsite.com/success",
    }
    
    jsonData, err := json.Marshal(paymentLink)
    if err != nil {
        return fmt.Errorf("error marshaling JSON: %v", err)
    }
    
    req, err := http.NewRequest("POST", url, bytes.NewBuffer(jsonData))
    if err != nil {
        return fmt.Errorf("error creating request: %v", err)
    }
    
    req.Header.Set("x-api-key", "lip_sk_live_YOUR_SECRET_KEY")
    req.Header.Set("Content-Type", "application/json")
    
    client := &http.Client{}
    resp, err := client.Do(req)
    if err != nil {
        return fmt.Errorf("error making request: %v", err)
    }
    defer resp.Body.Close()
    
    body, err := ioutil.ReadAll(resp.Body)
    if err != nil {
        return fmt.Errorf("error reading response: %v", err)
    }
    
    if resp.StatusCode == 200 || resp.StatusCode == 201 {
        var result map[string]interface{}
        json.Unmarshal(body, &result)
        fmt.Printf("Payment link created: %s\n", result["url"])
        return nil
    }
    
    return fmt.Errorf("error creating payment link: %s", string(body))
}

func main() {
    if err := createPaymentLink(); err != nil {
        fmt.Println(err)
    }
}

const app = express();

// Middleware to capture raw body for signature verification
app.use('/webhook', express.raw({ type: 'application/json' }));

// Webhook endpoint with signature verification
app.post('/webhook', (req, res) => {
  try {
    const signature = req.headers['x-lipana-signature'];
    
    if (!signature) {
      console.log('Missing X-Lipana-Signature header');
      return res.status(401).send('Unauthorized');
    }
    
    // Get raw body as string
    const payload = req.body.toString();
    
    // Verify signature
    const expectedSignature = crypto
      .createHmac('sha256', process.env.LIPANA_WEBHOOK_SECRET)
      .update(payload)
      .digest('hex');
    
    // Use constant-time comparison to prevent timing attacks
    if (!crypto.timingSafeEqual(
      Buffer.from(signature),
      Buffer.from(expectedSignature)
    )) {
      console.log('Invalid webhook signature');
      return res.status(401).send('Unauthorized');
    }
    
    // Parse the verified payload
    const data = JSON.parse(payload);
    const { event, data: eventData } = data;
    
    // Process webhook based on event type
    switch (event) {
      case 'transaction.success':
        console.log('Transaction successful:', eventData.transaction_id);
        // Update your database, send confirmation email, etc.
        break;
        
      case 'transaction.failed':
        console.log('Transaction failed:', eventData.transaction_id);
        // Handle failed transaction
        break;
        
      case 'transaction.cancelled':
        console.log('Transaction cancelled:', eventData.transaction_id);
        // Handle cancelled transaction
        break;
        
      default:
        console.log('Unknown event:', event);
    }
    
    res.status(200).json({ received: true });
  } catch (error) {
    console.error('Webhook processing error:', error);
    res.status(500).send('Internal Server Error');
  }
});

app.listen(3000, () => {
  console.log('Webhook server listening on port 3000');
});

Follow these best practices to ensure secure and reliable integration with Lipana.

Security
Store API Keys Securely

Never hardcode API keys in your source code. Always use environment variables or secure secret management systems.

Use Secret Keys Only on the Server

Secret keys (lip_sk_*) should only be used in server-side code. For client-side applications, use publishable keys (lip_pk_*) when available.

Verify Webhook Signatures

Always verify webhook signatures before processing webhook events. This ensures the request is authentic and hasn't been tampered with.

Never Commit Secrets to Version Control

Add .env to your .gitignore file and use environment-specific configuration files.

Error Handling
Handle API Errors Gracefully

Always implement proper error handling for API requests. Check HTTP status codes and handle different error scenarios appropriately.

Implement Retry Logic

For critical operations, implement exponential backoff retry logic for transient failures (network issues, rate limits, etc.).

Log Errors Appropriately

Log errors with sufficient context for debugging, but avoid logging sensitive information like API keys or customer data.

Performance
Use Webhooks Instead of Polling

Instead of repeatedly polling for transaction status, use webhooks to receive real-time notifications when events occur.

Implement Request Timeouts

Set appropriate timeout values for API requests to prevent your application from hanging on slow or unresponsive requests.

Cache Static Data

Cache frequently accessed but rarely changing data (like payment link details) to reduce API calls and improve response times.

Testing
Use Sandbox Environment

Always test your integration in the sandbox environment before going live. Use test API keys and test phone numbers.
